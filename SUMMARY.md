## Пользовательский опыт

### Сценарий 1: Новая карточка
1. Пользователь видит карточку со статусом "Новая"
2. Нажимает "Good" → карточка переходит в "Обучение" (10 мин)
3. Через 10 минут снова "Good" → переход в "Повторение" (1 день)
4. Через день "Good" → интервал 6 дней
5. Через 6 дней "Good" → интервал 15 дней (6 × 2.5)

### Сценарий 2: Забывание карточки
1. Карточка в состоянии "Повторение" (интервал 15 дней)
2. Пользователь нажимает "Again" → переход в "Переобучение" (10 мин)
3. EF уменьшается с 2.5 до 2.3
4. Через 10 минут "Good" → возврат в "Повторение" (1 день)
5. Дальше интервалы растут медленнее из-за сниженного EF

### Сценарий 3: Выбор колоды
1. Пользователь открывает приложение
2. Видит список колод:
   - "English Vocabulary" - 15 карточек (синяя рамка, бейдж "15")
   - "Python Programming" - ✓ Всё изучено
   - "Spanish Basics" - 8 карточек (синяя рамка, бейдж "8")
3. Видит панель "Готовы к изучению: 23 карточки из всех колод"
4. Может выбрать:
   - Нажать "Учить" на конкретной колоде
   - Нажать "Учить всё" для смешанного изучения

## Технические детали

### База данных
```sql
-- Новые поля в таблице progress
state VARCHAR  -- 'new', 'learning', 'review', 'relearning'
learning_step INTEGER  -- 0, 1, 2...
```

### API
```
GET /study/all/next  -- Следующая карточка из всех колод
GET /study/{deck_id}/next  -- Следующая карточка из конкретной колоды
POST /study/review?card_id={id}  -- Отправить оценку
```

### Формулы алгоритма
```
Again: EF -= 0.20, interval = 0 (переход в learning/relearning)
Hard: EF -= 0.15, interval *= 1.2
Good: EF += 0.15 (первые 3 раза), interval *= EF
Easy: EF += 0.15, interval *= EF * 1.3

EF ограничен: 1.3 ≤ EF ≤ 2.5
```
