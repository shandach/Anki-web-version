# Changelog - Логика выбора колод Anki

**Дата:** 2026-04-29

## Что было добавлено

### ✅ Реализована логика выбора колод как в Anki

Добавлена функциональность, полностью соответствующая логике работы Anki при открытии приложения.

## Основные изменения

### 1. Главный экран - список колод

**Изменено:**
- Главная страница (`/`) теперь редиректит на `/decks` (вместо `/login`)
- Список колод стал основным экраном приложения
- Пользователь сразу видит все свои колоды с количеством due карточек

**Почему:** В Anki пользователь не встречается с автоматической сессией повторения. Он видит список колод и сам выбирает, что изучать.

### 2. Улучшенное отображение due карточек

**Добавлено:**
- Визуальные индикаторы для колод с карточками к изучению
- Колоды с due карточками выделяются синей рамкой и фоном
- Количество due карточек отображается в синем бейдже в правом верхнем углу
- Статус "✓ Всё изучено" для колод без due карточек
- Кнопка "Учить" появляется только если есть карточки к изучению

**Визуальная иерархия:**
```
Колода с due карточками:
├── Синяя рамка (border-2 border-blue-300)
├── Градиентный фон (from-blue-50 to-white)
├── Синий бейдж с числом due карточек
└── Яркая кнопка "Учить"

Колода без due карточек:
├── Белый фон
├── Зеленая галочка "✓ Всё изучено"
└── Только кнопка "Открыть"
```

### 3. Функция "Учить всё"

**Новый endpoint:** `GET /study/all/next`
- Возвращает следующую карточку из всех колод пользователя
- Приоритизация по `due_date` (самые старые первыми)
- Смешивает карточки из разных колод в одну очередь

**Новая страница:** `/study/all`
- Изучение карточек из всех колод сразу
- Индикатор "Учить всё" в навигации
- Показывает состояние карточки и прогнозируемые интервалы

**UI элемент:**
- Большая синяя панель вверху списка колод
- Показывает общее количество due карточек
- Кнопка "Учить всё" для запуска смешанной сессии
- Появляется только если есть хотя бы одна due карточка

### 4. Улучшенный интерфейс изучения

**Добавлено в кнопки оценок:**
- Прогнозируемые интервалы под каждой кнопкой
  - Again: "<1 мин"
  - Hard: "~10 мин"
  - Good: "1 день" или расчет по формуле
  - Easy: "4 дня" или расчет по формуле

**Добавлено в футер карточки:**
- Отображение состояния карточки:
  - "Новая" (new)
  - "Обучение" (learning)
  - "Повторение" (review)
  - "Переобучение" (relearning)

**Формулы расчета интервалов:**
```typescript
Good: 
  - new/learning: 1 день
  - review: interval_days × ease_factor

Easy:
  - new/learning: 4 дня
  - review: interval_days × ease_factor × 1.3
```

## Пользовательский опыт

### Сценарий 1: Открытие приложения
1. Пользователь открывает приложение
2. Видит список всех своих колод
3. Колоды с due карточками выделены синим
4. Видит общее количество карточек к изучению
5. Может выбрать:
   - Конкретную колоду → изучать только её
   - "Учить всё" → изучать карточки из всех колод

### Сценарий 2: Целенаправленное изучение
1. Пользователь видит, что в колоде "English Vocabulary" 15 карточек
2. Нажимает "Учить" на этой колоде
3. Изучает только английские слова
4. После завершения возвращается к списку колод

### Сценарий 3: Смешанное изучение
1. Пользователь видит панель "Готовы к изучению: 42 карточки"
2. Нажимает "Учить всё"
3. Карточки из разных колод показываются вперемешку
4. Приоритет отдается самым старым (по due_date)

### Сценарий 4: Всё изучено
1. Пользователь завершил все карточки
2. Видит "✓ Всё изучено" на всех колодах
3. Панель "Учить всё" не отображается
4. Может добавить новые карточки или отдохнуть

## Преимущества

✅ **Контроль:** пользователь сам решает, что изучать
✅ **Прозрачность:** видно количество due карточек для каждой колоды
✅ **Гибкость:** можно изучать одну колоду или все сразу
✅ **Визуальная ясность:** колоды с работой выделены, без работы - спокойные
✅ **Мотивация:** видно прогресс ("✓ Всё изучено")
✅ **Информативность:** показаны прогнозируемые интервалы и состояния

## Технические детали

### Endpoint `/study/all/next`

```python
# Приоритизация карточек:
1. Due карточки (due_date <= today)
   - Сортировка по due_date ASC (старые первыми)
2. Новые карточки (без progress)
   - Любая новая карточка из любой колоды
```

### Визуальная иерархия колод

```typescript
Priority 1: Колоды с due карточками
  - border-2 border-blue-300
  - bg-gradient-to-br from-blue-50 to-white
  - Синий бейдж с числом

Priority 2: Quiz колоды
  - border-2 border-purple-200
  - bg-gradient-to-br from-purple-50 to-white
  - Фиолетовый бейдж "QUIZ"

Priority 3: Обычные колоды
  - bg-white
  - Без специальных индикаторов
```
